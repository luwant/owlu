# 已实现总览（截至 2026-03-11）

## 架构概述

OWLU (Open World Label Updating) 采用**三模块闭环架构**:

```
Discovery ──MatchResult──▶ Writer ──promoted_labels──▶ Absorption
    ▲                                                       │
    └───────────── 更新后的 model_state (E, P, threshold) ◀─┘
```

| 模块 | 职责 | 状态 | 文档 |
|------|------|------|------|
| **Candidate Discovery** | 从低置信样本发现候选标签 | ✅ 已实现 | [candidate_discovery.md](candidate_discovery.md) |
| **Ontology-Constrained Writer** | 跨文档积累 + 本体约束晋升 | ✅ 已实现 | [ontology_writer.md](ontology_writer.md) |
| **Prototype Absorption** | 将新标签吸收到 LTC-MPE 表示空间 | ⚡ fast: 已实现+验证; slow: dict 模式 | [prototype_absorption.md](prototype_absorption.md) |

## Module 1: Candidate Discovery

从"模型目前解释不好的样本"里，发现值得进入闭环体系审核的候选标签。

- **LtceGate** (`discovery/gate.py`):
  基于 `raw_max_prob`（max sigmoid at T=1.0）判定 OOD，自适应校准（验证集第 5 百分位 + floor=0.10）。
- **LLMPhraseGenerator** (`discovery/phrase_generator.py`):
  DeepSeek 兼容 API 调用 + JSON 容错解析 + 多采样一致性投票（k=3, agreement）。
- **SemanticMatcher** (`discovery/matcher.py`):
  双模式：默认 BOW cosine 轻量匹配（零依赖）；注入 `dense_encoder` 后切换为稠密向量匹配（SentenceTransformer），带标签嵌入缓存。三路初判（merge_pre / novel_pre / hold_pre）。
- **CandidateDiscovery** (`discovery/__init__.py`):
  Facade，一步完成 gate→extract→match。

**输出**: `list[MatchResult]` → 送入 Writer。

详见 [candidate_discovery.md](candidate_discovery.md)。

---

## Module 2: Ontology-Constrained Writer

把候选标签写入标签体系（以 AAPD 为例），在写入前进行本体约束检查。

- **LabelBank** (`writer/label_bank.py`):
  状态机 hold → candidate → promoted，跨文档簇聚合，四重门控晋升（freq ≥ 3, source_docs ≥ 2, agreement ≥ 0.5, semantic_distance ≥ 0.3）。短语归一化去标点并合并缩写（如 N.L.P. → nlp）。
- **OntologyConstraintChecker** (`writer/constraints.py`):
  命名格式校验 + 父类存在性检查 + 重复标签拒绝。提供 `for_aapd()` 工厂方法。
- **LabelBankStore** (`writer/persistence.py`):
  SQLite 持久化层，6 张表存储标签、别名、簇、短语、来源文档，支持 `save()` / `load()` 全量序列化。
- **OntologyWriter** (`writer/__init__.py`):
  Facade，ingest + promote + 标签清单导出 + 可选 `db_path` 自动持久化 + `from_db()` 恢复。

**输出**: `promoted_labels` + `LabelBank` 实例 → 送入 Absorption。

详见 [ontology_writer.md](ontology_writer.md)。

## Module 3: Prototype Absorption

把已经被闭环体系批准的新标签，真正吸收到 LTC-MPE 的表示空间里。

- **metrics.py** (`absorption/metrics.py`):
  双层实现：纯 Python 向量运算 + Torch 加速工具。`recalibrate_model_threshold` 基于实际模型 forward pass 的 sigmoid(logits) 做 grid search。
- **fast_sync / fast_sync_model** (`absorption/fast_sync.py`):
  别名语义混合 E/P（η_e=0.2, η_p=0.1）+ 阈值重标定，标签数不变。
  - `fast_sync` — dict 模式（纯 Python）
  - `fast_sync_model` — **model 模式（Torch，直接操作 LTCEModel 注册缓冲区，已在 AAPD 上验证 ✅）**
- **slow_sync** (`absorption/slow_sync.py`):
  标签扩维 L→L'，新标签初始化 + 一次增量微调 + 阈值重标定。仅 dict 模式，尚未对接 LTCE 模型增量训练。
- **PrototypeAbsorption** (`absorption/__init__.py`):
  Facade，`fast_absorb()` / `slow_absorb()` (dict) + `fast_absorb_model()` (model) 三种吸收策略。

**fast_sync_model 验证结果** (AAPD, 10 标签别名注入):
- Macro-F1 +0.011 (0.584→0.595), Micro-F1 -0.005 (0.742→0.737)
- E 向量漂移 cosine ≈ 0.9999, P 漂移 ≈ 0.994

**输出**: 更新后的 `model_state {E, P, threshold}` → 反馈给 Discovery。

详见 [prototype_absorption.md](prototype_absorption.md)。

## 共享数据结构 (common/types.py)

| 结构 | 说明 |
|------|------|
| `OWLUConfig` | 统一配置，从 owlu.yaml 加载（含 LLM / 阈值 / 门控 / 采样参数） |
| `CandidatePhrase` | LLM 产出的候选短语（text, agreement, source_doc_id, ...） |
| `MatchResult` | 语义匹配结果（action, similarity, target_label） |
| `LabelInfo` | 标签元数据（aliases, description） |
| `ProtoLabelCluster` | 候选簇（phrases, freq, source_docs, state） |
| `GateDecision` | 门控决策（should_invoke_llm, raw_max_prob） |
| `ValidationSample` | 验证样本（embedding, true_labels） |
