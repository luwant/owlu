# 已实现总览（截至 2026-03-13）

## 架构概述

OWLU 采用三模块闭环：

```text
Discovery ──MatchResult──▶ Writer ──promoted_labels──▶ Absorption
    ▲                                                       │
    └──────────── 更新后的 LTCE / model_state ◀─────────────┘
```

| 模块 | 职责 | 状态 | 文档 |
|------|------|------|------|
| Candidate Discovery | 从低置信样本发现候选标签 | 已实现 | [candidate_discovery.md](candidate_discovery.md) |
| Ontology-Constrained Writer | 跨文档积累、约束检查、晋升新标签 | 已实现 | [ontology_writer.md](ontology_writer.md) |
| Prototype Absorption | 把标签知识吸收到 LTCE 表示空间 | fast 已验证；slow 已对接真实 LTCE | [prototype_absorption.md](prototype_absorption.md) |

## Module 1: Candidate Discovery

Candidate Discovery 的职责是从 LTCE 低置信样本里发现候选标签。

- `LtceGate`：
  基于 `raw_max_prob` 做门控，决定是否调用 LLM。
- `LLMPhraseGenerator`：
  DeepSeek 兼容调用、JSON 容错、多次采样一致性投票。
- `SemanticMatcher`：
  支持 BOW 和 dense 两种语义匹配模式。
- `CandidateDiscovery`：
  facade，一步完成 gate -> extract -> match。

输出为 `list[MatchResult]`，进入 Writer。

---

## Module 2: Ontology-Constrained Writer

Writer 负责把候选短语稳定写入标签体系。

- `LabelBank`：
  维护 hold / candidate / promoted 三态，执行四重门控晋升。
- `OntologyConstraintChecker`：
  做命名、父类、重复标签等约束检查。
- `LabelBankStore`：
  SQLite 持久化。
- `OntologyWriter`：
  facade，支持 ingest、promote、save、load。

输出为：

- `LabelBank`
- `promoted_labels`

这两者都会被 Absorption 使用。

---

## Module 3: Prototype Absorption

Absorption 负责把闭环批准后的标签真正写回 LTCE。

### 已实现能力

- `metrics.py`
  - 纯 Python 向量运算与阈值校准
  - Torch 阈值重标定与向量混合
- `fast_sync / fast_sync_model`
  - 别名语义混合到 `E / P`
  - 阈值重标定
  - `fast_sync_model` 已在 AAPD alias 注入实验验证
- `slow_sync / slow_sync_model`
  - 标签空间从 `L` 扩到 `L'`
  - 新标签 `label_embeddings / label_prototypes` 初始化
  - `classifier[-1]` 输出维度扩展
  - 增量微调
  - 阈值重标定
- `ltce_bridge.py`
  - 从 `Label-gen` 加载真实 LTCE config / checkpoint / tokenizer / dataloader
  - 构造 expanded loader，使 `labels.shape[1] == L'`
- `PrototypeAbsorption`
  - facade
  - 统一提供 dict 模式、model 模式和 LTCE bridge 入口

### 当前状态

- `fast_sync_model`：已在 AAPD 上做过 alias 注入验证
- `slow_sync_model`：已不再是“仅 dict 模式”
- 真实 LTCE 对接路径已经打通：
  - 可加载 `Label-gen/configs/ltce_aapd_full.yaml`
  - 可加载 `best.pt`
  - 可构造 AAPD train / val / test loader
  - 可将标签维度从 54 扩到 55 并产出 expanded batch

### 仍未固化的内容

- 仓库内尚未附带完整 AAPD slow-sync 指标实验报告
- 新标签效果仍依赖真实正例文本或旧文档标签补充映射
