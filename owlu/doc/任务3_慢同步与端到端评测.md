# 任务3：慢同步与端到端评测

## 1. 任务目标

完成 OWLU 后半链路，覆盖：

- 候选簇晋升与标签扩容（`L -> L'`）
- 慢同步增量微调
- 端到端结果验证与消融实验

该任务交付后，应完成“周期扩容 + 增量微调 + 指标评测”全闭环。

---

## 2. 开发范围

### 2.1 代码范围

- `owlu/ltc_sync.py`（`slow_sync`）
- 评测脚本（建议放在 `owlu/` 或现有 `scripts/`）

### 2.2 慢同步关键流程

1. 周期末从 `proto_label_clusters` 读取审核通过簇。
2. 扩展标签集 `L -> L'`，初始化新增标签 embedding/prototype。
3. 旧样本回放 + 新标签样本做一次增量微调。
4. 重新评估并更新发布阈值。

说明：该流程是离线周期扩容，不属于在线热扩容。

旧样本回放策略（固定）：

- 采样方式：分层回放（按旧标签分层）。
- 每个旧标签最多采样：
  - 正样本 `max_pos_per_label = 30`
  - 负样本 `max_neg_per_label = 30`
- 新旧训练比例：`old:new = 2:1`（按样本数）。
- 随机种子固定：`seed=42`，保证可复现。

发布阈值重标定策略（固定）：

- 在验证集上网格搜索阈值 `t ∈ [0.10, 0.90]`，步长 `0.02`。
- 目标：优先最大化 `Macro-F1`。
- 约束：`P@3` 相比慢同步前下降不超过 `1.0` 绝对点。

### 2.3 端到端评测范围

- 主链路：
  - 外部文本 -> 抽取 -> 门控 -> 标签库 -> 快/慢同步
- 指标：
  - 覆盖率
  - Micro/Macro-F1
  - 长尾标签 F1
  - 候选通过率
  - token 成本与时延
