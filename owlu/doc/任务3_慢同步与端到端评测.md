# 任务3：慢同步与端到端评测（开发计划 v2）

## 1. 目标与边界

本任务目标是补齐 OWLU 后半闭环：`标签扩容(L -> L') + 慢同步增量微调 + 端到端评测/消融`。

本任务是离线周期流程，不做在线热扩容，不改变 Task1/Task2 已通过逻辑。
主存储统一使用 `SQLite`。

---

## 2. 与当前实现对齐后的主方向

### 2.1 数据来源修正

- 慢同步输入的“审核通过簇”来源应为 `LabelBank.promoted_labels`（而非 `proto_label_clusters`）。
- `proto_label_clusters` 继续作为过程态池；晋升后数据以 `promoted_labels` 为准。

### 2.2 `slow_sync` 核心职责

在 `owlu/ltc_sync.py` 新增 `slow_sync(...)`，职责如下：

1. 读取已晋升簇并生成新增标签 ID 列表。
2. 扩展模型标签空间（`L -> L'`），初始化新增标签 `E/P`。
3. 使用“旧样本回放 + 新标签样本”做一次增量微调。
4. 在验证集重标定阈值并满足发布约束。
5. 输出可追踪的 `sync_report`（包含扩容数量、训练配置、阈值变化、关键指标）。

### 2.3 扩维与初始化原则

- 新标签初始化优先使用 `canonical + aliases` 的文本编码均值。
- 若语义文本不足，回退到簇代表短语编码。
- 旧标签参数尽量保持不变，仅在增量训练阶段更新。
- 保证可回滚：保留慢同步前模型快照与阈值。

### 2.4 回放与增量训练固定策略

- 旧样本回放采用按标签分层采样。
- 每个旧标签上限：`max_pos_per_label = 30`，`max_neg_per_label = 30`。
- 样本比例：`old:new = 2:1`。
- 随机种子：`seed = 42`。

### 2.5 阈值重标定固定策略

- 网格范围：`t ∈ [0.10, 0.90]`，步长 `0.02`。
- 主目标：最大化 `Macro-F1`。
- 约束：`P@3` 相比慢同步前下降不超过 `1.0` 绝对点。
- 若多个阈值同分，优先离旧阈值更近者。

### 2.6 主存储方案（SQLite）

- 使用 `SQLite` 作为任务3主存储，面向单机离线流程与可复现实验。
- 至少落盘以下信息：
  - 标签与别名快照（支持慢同步前后对比）。
  - 已晋升簇（`promoted_labels`）及其证据信息。
  - 慢同步运行记录（时间、配置、扩容数量、阈值变化、核心指标）。
  - 端到端评测与消融结果摘要。
- 要求：
  - 支持事务写入，避免一次慢同步过程中的中间态不一致。
  - 关键表保留 `run_id` 或时间戳字段，保证可追溯。
  - 提供最小迁移脚本或初始化脚本，支持空库一键建表。

## 3. 里程碑（建议顺序）

### M1：慢同步输入与扩维骨架

- 完成 `slow_sync` 输入契约与 `promoted_labels` 接入。
- 打通 `L -> L'` 的标签扩容、参数初始化、模型重建。
- 建立 SQLite 初始化与基础读写层（至少覆盖晋升簇读取、运行记录写入）。
- 先保证可运行，再加训练与评测。

### M2：增量训练与发布阈值

- 实现回放采样器与固定比例策略。
- 完成一次增量微调流程。
- 完成带 `P@3` 约束的阈值重标定。

### M3：端到端评测与消融

- 新增任务3评测脚本（建议 `owlu/` 或 `scripts/`）。
- 输出统一指标报告：覆盖率、Micro/Macro-F1、长尾 F1、候选通过率、token 成本、时延。
- 完成 3 组消融：去一致性门控、去不确定性回访、去候选簇晋升。
- 评测与消融结果写入 SQLite，并可按 `run_id` 回查。

- 慢同步流程可稳定运行，且支持标签扩容。

### 7.2 与原步骤存在差异的点（必须阅读）

- “增量微调”当前采用**轻量一阶近似**（基于样本质心的 `E/P` 更新），而非完整梯度训练循环。  
  说明：该实现用于保证任务3在当前仓库中可稳定跑通、可单测、可回放；后续若进入论文/生产级实验，可替换为真实训练器。
- `P@3` 约束按**百分比点**实现：`P@3 ∈ [0,100]`，`max_p_at_3_drop=1.0` 表示最多下降 1 个百分点。
- SQLite 当前记录的是最小可追溯闭环（标签快照、晋升簇、slow_sync 记录、e2e/ablation 报告）；未扩展到完整实验资产归档（如模型权重文件索引）。
