# 任务2：标签库与快同步开发

## 1. 任务目标

完成 OWLU 中段能力，覆盖：

- 跨文档状态累积与最终判定（candidate / hold / cluster）
- 审核包生成
- 与 LTC-MPE 的快同步（语义级，不扩维，含 prototype 对齐）

该任务交付后，应可在不改变 `num_labels` 的前提下完成标签语义更新并保持推理兼容。

---

## 2. 开发范围

### 2.1 代码范围

- `owlu/label_bank.py`
- `owlu/ltc_sync.py`（仅 `fast_sync`）

### 2.2 关键能力

- `LabelBank` 支持：
  - `candidate_labels`
  - `hold_pool`
  - `proto_label_clusters`
- 接收 Task 1 的 `merge_pre/novel_pre/hold_pre` 结果并做最终动作判定
- 统计字段：
  - `freq`
  - `source_docs`
  - `agreement`
- 最终判定规则：
  - `candidate`: `novel_pre` 且 `freq >= 3` 且 `source_docs >= 2`
  - `hold`: 其余未达晋升条件的 `novel_pre/hold_pre`
  - `merge`: 对 `merge_pre` 直接并入 alias/description
- 审核包输出：
  - 代表短语
  - 证据文档
  - 最近已有标签距离

### 2.3 快同步约束

- 只做语义级更新：
  - 刷新 `aliases/description`
  - 重编码标签向量 `E`
  - 对齐更新 prototype buffer `P`
  - 重新校准阈值
- 不改变标签维度，不重建分类头。

推荐更新公式：

\[
e'_y=\mathrm{Norm}\big((1-\eta_e)e_y+\eta_e\bar e_{alias}\big)
\]
\[
p'_y=\mathrm{Norm}\big((1-\eta_p)p_y+\eta_p e'_y\big)
\]

默认系数：`eta_e=0.2`, `eta_p=0.1`。
