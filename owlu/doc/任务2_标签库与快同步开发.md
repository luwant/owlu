## 2. LabelBank 具体实现

## 2.1 核心数据结构

`LabelBank` 内部维护 5 类状态：

- `labels: dict[str, LabelInfo]`  
  已有标签的可变语义信息（`aliases`、`description`）。
- `proto_label_clusters: dict[str, ProtoLabelCluster]`  
  所有候选/观察簇的总表。
- `candidate_labels: dict[str, ProtoLabelCluster]`  
  达到晋升条件的候选簇。
- `hold_pool: dict[str, ProtoLabelCluster]`  
  未达晋升条件的观察簇。
- `promoted_labels: dict[str, ProtoLabelCluster]`  
  已执行晋升动作的簇记录。

`ProtoLabelCluster` 字段：

- `cluster_id`
- `representative_phrase`
- `phrases: dict[phrase, count]`
- `source_docs` / `evidence_docs`
- `freq`
- `agreement_sum` / `agreement_count` / `agreement`（属性）
- `nearest_label_id`
- `nearest_label_distance`
- `state: hold|candidate|promoted`

## 2.2 最终动作判定与状态转移

`process_match_result(MatchResult)` 对接 Task1 输出：

- `merge_pre`：直接调用 `add_alias` 并补充 `description`。
- `novel_pre`：进入 `add_candidate`；满足晋升条件转 `candidate`，否则放入 `hold_pool`。
- `hold_pre`：进入 `add_hold`。

当前晋升阈值（默认）：

- `min_freq = 3`
- `min_source_docs = 2`

即满足：

- `freq >= 3` 且 `source_docs >= 2` -> `candidate`
- 否则 -> `hold`

## 2.3 proto_label_clusters 聚簇策略（当前实现）

当前聚簇是轻量策略，不是复杂语义聚类：

- 默认 `cluster_id = normalize(phrase_text)`（小写+空白规整）
- 支持外部显式传入 `cluster_id`，可将同义短语强制归并
- `representative_phrase` 通过簇内短语计数投票确定（高频优先）
- `nearest_label_distance` 保留历史最小值（更近的标签关系）

这使得 `proto_label_clusters` 可用于审核和慢同步前晋升，但语义聚簇能力仍偏规则化。

## 2.4 审核包与晋升

已支持：

- `summarize_cluster(cluster_id)`：输出簇摘要
- `build_review_packet(cluster_id)`：输出审核包字段
  - `representative_phrase`
  - `evidence_docs`
  - `nearest_label_distance`
  - `freq`
  - `source_docs`
  - `agreement`
- `promote_cluster(cluster_id, new_label_id)`：仅允许从 `candidate` 晋升，晋升后写入 `labels` 并从候选/观察池摘除

## 3. fast_sync 具体实现

## 3.1 输入输出接口

输入 `model_state`（字典协议）：

- `label_ids`
- `E`（标签嵌入）
- `P`（prototype buffer）
- `threshold`

输出更新后的 `model_state`，附加：

- `label_aliases`
- `label_descriptions`
- `sync_report`

## 3.2 语义更新流程

每个标签执行：

1. 从 `LabelBank` 取 `aliases`；为空则回退到 `label_id` 文本。
2. 对 alias 文本编码并求均值 `alias_mean`。
3. 更新 `E`：

\[
e'_y=\mathrm{Norm}((1-\eta_e)e_y+\eta_e\bar e_{alias})
\]

4. 更新 `P`：

\[
p'_y=\mathrm{Norm}((1-\eta_p)p_y+\eta_p e'_y)
\]

默认系数：`eta_e=0.2`, `eta_p=0.1`。

## 3.3 阈值重标定

`recalibrate_threshold` 在验证集做网格搜索：

- 阈值区间：`[0.10, 0.90]`
- 步长：`0.02`
- 目标：最大化 `Macro-F1`
- 平分处理：优先选择离旧阈值更近的点

## 3.4 维度稳定性约束

`fast_sync` 强校验：

- `len(label_ids) == len(E) == len(P)`
- `E/P` 行维度一致
- 不新增标签、不改 `classifier out_features`

因此满足“快同步不扩维”的工程约束。

说明：该结果证明“有可观测变化”，但样本量较小，不能视为最终结论。

## 5. 可用性与学术价值评估

## 5.1 工程可用性（结论：可用）

在任务2目标下，当前实现已达到可用：

- 状态机完整（`merge`/`hold`/`candidate`/`promoted`）
- 快同步可对真实 LTCE 模型执行并保持维度稳定
- 阈值可自动重标定
- 已有单测、集成测、OOD 对比测三层验证

## 5.2 学术价值（结论：有潜力，但尚未到论文级充分证据）

已有价值：

- 给出“标签语义刷新 + prototype 对齐 + 阈值重标定”的可复现实装闭环
- 形成了可用于进一步实验的基础平台

当前不足：

- `proto_label_clusters` 仍是轻量规则聚簇，未引入更强的语义聚类/图聚类方法
- OOD 评测样本规模小，且为手工构造，统计显著性不足
- 目前 OOD 验证未闭环接入 Task1 的在线 LLM 抽取结果（主要验证 fast_sync 能力）
- 当前实现未包含论文级消融矩阵、显著性检验、多数据集复现

结论：

- 工程落地层面：已可用
- 学术发表层面：还需要系统实验与方法增强

## 6. 当前边界与后续建议

建议后续优先补齐：

1. 语义级聚簇升级：从“短语归一化聚簇”升级到 embedding/图聚类。  
2. 评测升级：扩大 OOD 样本规模，加入真实外部语料分布偏移测试。  
3. 闭环升级：将 Task1 LLM 输出自动流入 LabelBank，再触发快同步。  
4. 研究升级：补充系统消融、置信区间、显著性检验与多种基线对比。
